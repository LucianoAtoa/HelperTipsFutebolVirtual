<decisions>
## Implementation Decisions

### Armazenamento da config de mercados
- **D-01:** Tabela PostgreSQL para mercados e suas complementares — não arquivo JSON
- **D-02:** Estrutura: tabela `mercados` (id, slug, nome_display, odd_ref, ativo) + tabela `complementares` (id, mercado_id, slug, nome_display, percentual, odd_ref, regra_validacao)
- **D-03:** Editável sem mexer no código — futuro CRUD no dashboard possível

### Parser e captura
- **D-04:** Parser NÃO muda — o grupo envia apenas sinais do mercado principal (Over 2.5, Ambas Marcam)
- **D-05:** Complementares são apostas paralelas criadas pelo usuário, calculadas como % da stake principal
- **D-06:** Validação das complementares é feita automaticamente a partir do placar do sinal principal

### Lógica de validação por placar
- **D-07:** Placar `X-Y` do sinal = `casa-fora`. Dado placar, cada complementar é validada:
  - Over 3.5 → GREEN se `casa + fora > 3.5`
  - Over 5+ → GREEN se `casa + fora >= 6`
  - Empate 3-3 / 4-4 → GREEN se placar é exatamente `3-3` ou `4-4`
  - Total Gols Casa = 4 → GREEN se `casa == 4`
  - Total Gols Fora = 4 → GREEN se `fora == 4`
  - Total Gols Casa 5+ → GREEN se `casa >= 5`
  - Total Gols Fora 5+ → GREEN se `fora >= 5`
- **D-08:** Sinal RED (sem placar) → todas complementares RED automaticamente
- **D-09:** Sinal pendente (sem resultado) → complementares pendentes também
- **D-10:** Melhoria futura: capturar placar dos sinais RED quando o bot passar a informar

### Cálculo de stakes Martingale
- **D-11:** Stake complementar = percentual × stake principal da tentativa atual
- **D-12:** Complementares dobram junto com a principal (fator 2x por tentativa)
- **D-13:** Investimento total por tentativa = principal + soma das complementares
- **D-14:** Fórmula: `stake_complementar_TN = stake_inicial × (2^(N-1)) × percentual`

### ROI no dashboard
- **D-15:** Dois cards separados: "ROI Principal" (existente, atualizado) e "ROI Complementares" (novo)
- **D-16:** Card complementares mostra breakdown por mercado (cada linha: mercado, greens, reds, lucro)
- **D-17:** ROI total considera investimento principal + todas complementares

### Configuração JSON de referência (seed data para popular as tabelas)
- **D-18:** Dois mercados principais iniciais: Over 2.5 (odd ref 2.30) e Ambas Marcam (odd ref 2.10)
- **D-19:** Over 2.5 tem 7 complementares totalizando 35% da stake principal
- **D-20:** Ambas Marcam tem 7 complementares totalizando 20% da stake principal
- **D-21:** Percentuais e odds devem ser ajustáveis — a tabela PostgreSQL permite UPDATE direto

## Specific Ideas

- O placar já é capturado pelo parser como string "X-Y" — basta split e int() para validar complementares
- A tabela `complementares` pode ter campo `regra_validacao` com expressão avaliável (ex: "total > 3.5", "casa == 4")
- Seed data vem direto da documentação do usuário — 2 mercados × 7 complementares = 14 registros
- Futebol virtual Bet365 roda 24/7 com RNG — odds são estáveis, referências da documentação são confiáveis
- ~41% das partidas terminam Over 2.5 (baseado em 667k partidas) — baseline estatístico útil

</specifics>
