## Spacing Scale

Declarado em termos de classes Bootstrap 5 (DARKLY) e valores px equivalentes.
Escala 8-point: todos múltiplos de 4.

| Token | Classe Bootstrap | Valor px | Uso |
|-------|-----------------|----------|-----|
| xs | `g-1`, `p-1` | 4px | Gaps de ícone, padding inline interno |
| sm | `g-2`, `p-2` | 8px | Espaçamento entre elementos compactos |
| md | `g-3`, `p-3` | 16px | Espaçamento padrão de elementos (estabelecido na Phase 02) |
| lg | `mb-3`, `py-3` | 24px | Padding de seção, separação entre cards |
| xl | `mb-4` | 32px | Gaps de layout entre blocos maiores |
| 2xl | `my-4` | 48px | Quebras de seção maiores |
| 3xl | n/a | 64px | Espaçamento de nível de página (não usado no dashboard) |

Exceções: nenhuma. Card ROI Complementares usa o mesmo padrão `mb-3` dos cards existentes.

**Fonte:** Padrão estabelecido na Phase 02 — `className="mb-3 g-2"` nas Rows do dashboard existente.

## Typography

Dash/Bootstrap DARKLY usa `system-ui, -apple-system, sans-serif`. Não há fontes externas.

| Role | Classe Bootstrap | Tamanho equiv. | Weight | Line Height |
|------|-----------------|----------------|--------|-------------|
| Body / tabela | padrão (`p`, `small`) | 14px | 400 (regular) | 1.5 |
| Label / card subtitle | `H6`, `card-title text-muted` | 14px | 400 (regular) | 1.5 |
| Valor KPI / ROI | `H3`, `H5 fw-bold` | 20px / 16px | 600 (semibold) | 1.2 |
| Heading de seção | `H5 card-title text-muted` | 16px | 400 (regular) | 1.2 |

Regras:
- Exatamente **2 pesos**: 400 (regular) para labels/títulos de card, 600 (fw-bold) para valores KPI e resultados monetários.
- Exatamente **3 tamanhos**: 14px (labels e tabela), 16px (headings de card e valores ROI secundários), 20px (valores KPI principais).
- Tamanho 28px (Display) não usado nesta fase — dashboard não tem título hero de grande porte.

**Fonte:** Padrão estabelecido na Phase 02 — `html.H6`, `html.H3`, `html.H5` + `fw-bold` no dashboard existente.

## Color

Tema base: Bootstrap 5 DARKLY. Cores semânticas mapeadas para as classes Bootstrap estabelecidas.

| Role | Valor hex | Classe Bootstrap | Uso |
|------|-----------|-----------------|-----|
| Dominant (60%) | `#222222` / `#303030` | fundo DARKLY padrão | Background da página, fundo de cards |
| Secondary (30%) | `#2a2a2a` / `#3a3a3a` | `dbc.Card` DARKLY | Cards, painéis, container do card ROI Complementares |
| Accent (10%) | `#17a2b8` | `text-info` | Percentuais de complementares, valores neutros de ROI |
| Green / sucesso | `#28a745` | `text-success` | Resultados GREEN, lucros positivos |
| Red / danger | `#dc3545` | `text-danger` | Resultados RED, prejuízos, valores negativos |
| Warning / pendente | `#ffc107` | `text-warning` | Sinais pendentes (estabelecido: rowStyle amber na tabela) |
| Muted | `#6c757d` | `text-muted` | Títulos de card, labels, totais secundários |

Accent (`#17a2b8`, `text-info`) reservado exclusivamente para:
- Percentual de cada complementar na tabela de breakdown (ex: "20%")
- Valor de "Total Investido" quando neutro (nem lucro nem prejuízo)
- Valores de ROI % quando exatamente 0.0%

Destructive: `#dc3545` (`text-danger`) reservado para valores RED, prejuízos e qualquer dado que representa perda. Nenhuma ação destrutiva de UI nesta fase (sem delete/reset de dados).

**Fonte:** Cores extraídas diretamente de `dashboard.py` — `marker_color=["#28a745", "#dc3545", "#ffc107"]`, `#17a2b8`, rowStyle `#3a3000/#ffc107`.

## Componentes Específicos da Fase

Esta seção descreve os contratos visuais dos componentes novos desta fase.

### Card ROI Complementares (novo)

Segue exatamente o padrão do card "Simulacao de ROI" existente.

**Estrutura:**
```
dbc.Card
  dbc.CardBody
    html.H5("ROI Complementares", className="card-title text-muted mb-3")
    dbc.Row [g-2]
      Col md=2: Label "Mercado" / Valor de seleção (controlado pelo filter-entrada)
      Col md=2: Label "Lucro/Prejuízo" / H5 fw-bold (cor dinâmica: text-success ou text-danger)
      Col md=2: Label "ROI %" / H5 fw-bold (cor dinâmica)
      Col md=2: Label "Total Investido" / H5 fw-bold text-muted
    [separador: hr ou mb-3]
    dbc.Table [tabela de breakdown por mercado complementar]
```

**Tabela de breakdown por mercado:**
- Componente: `dbc.Table` (NÃO AG Grid — 7 linhas fixas não justificam grid pesado)
- Colunas: Mercado | Greens | Reds | Lucro
- Largura: 100% do card
- Alinhamento: Mercado = left; Greens, Reds, Lucro = right
- Cor do valor Lucro: `text-success` se positivo, `text-danger` se negativo, `text-muted` se zero
- Última linha: totais em negrito (`fw-bold`)
- `size="sm"` para densidade compacta (Bootstrap Table prop)
- `bordered=False`, `hover=True`, `dark=True` (consistente com tema DARKLY)

**Estado vazio — sem filtro de entrada selecionado:**
Ver seção Copywriting abaixo. Exibir `dbc.Alert` em vez da tabela.

## Interaction Contract

### Comportamento do card complementares em função dos filtros

| Estado do filtro `entrada` | Comportamento do card |
|---------------------------|----------------------|
| `None` (sem filtro) | Exibe `dbc.Alert` com mensagem "Selecione uma entrada" (cor: `color="info"`) |
| "Over 2.5" | Carrega complementares de `over_2_5`, exibe tabela com 7 linhas |
| "Ambas Marcam" | Carrega complementares de `ambas_marcam`, exibe tabela com 7 linhas |
| Qualquer outro valor | Exibe `dbc.Alert` com mensagem "Sem complementares configuradas para esta entrada." |

### Colorização dinâmica dos valores monetários

| Condição | Classe CSS |
|----------|-----------|
| `profit > 0` | `text-success` |
| `profit < 0` | `text-danger` |
| `profit == 0` ou pendente | `text-muted` |
| `roi_pct > 0` | `text-success` |
| `roi_pct < 0` | `text-danger` |
| `roi_pct == 0` | `text-info` |

### Responsividade

Manter `fluid=True` e colunas `md=2` / `md=12` para mobile. Card complementares usa a mesma grid do ROI card principal. Em telas < md, colunas empilham verticalmente (comportamento Bootstrap padrão).
