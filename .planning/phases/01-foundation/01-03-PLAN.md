2. Create `store.py`:
   - NO imports from `telethon`, `parser`, or `asyncio` — this is a sync-only module
   - Import `psycopg2` (only for type hints if needed)

   - `def upsert_signal(conn, data: dict) -> None`:
     ```python
     with conn.cursor() as cur:
         cur.execute("""
             INSERT INTO signals (
                 message_id, liga, entrada, horario, periodo,
                 dia_semana, resultado, placar, raw_text, received_at, updated_at
             ) VALUES (
                 %(message_id)s, %(liga)s, %(entrada)s, %(horario)s, %(periodo)s,
                 %(dia_semana)s, %(resultado)s, %(placar)s, %(raw_text)s,
                 NOW(), NOW()
             )
             ON CONFLICT (message_id) DO UPDATE SET
                 resultado  = EXCLUDED.resultado,
                 placar     = EXCLUDED.placar,
                 liga       = EXCLUDED.liga,
                 entrada    = EXCLUDED.entrada,
                 updated_at = NOW()
             WHERE
                 signals.resultado IS DISTINCT FROM EXCLUDED.resultado
                 OR signals.resultado IS NULL
         """, data)
     conn.commit()
     ```

   - `def get_stats(conn) -> tuple`:
     ```python
     with conn.cursor() as cur:
         cur.execute("""
             SELECT
                 COUNT(*) AS total,
                 COUNT(*) FILTER (WHERE resultado = 'GREEN') AS greens,
                 COUNT(*) FILTER (WHERE resultado = 'RED')   AS reds,
                 COUNT(*) FILTER (WHERE resultado IS NULL)   AS pending
             FROM signals
         """)
         return cur.fetchone()
     ```

3. Key design points:
   - `upsert_signal` takes a raw psycopg2 connection (NOT async) — the caller (listener.py) is responsible for wrapping in `asyncio.to_thread(upsert_signal, conn, data)` per DB-04
   - The ON CONFLICT WHERE clause prevents overwriting a known resultado with NULL — if a signal already has GREEN/RED, re-processing the original message won't clear the result
   - `conn.commit()` is called inside `upsert_signal` — each write is its own transaction
