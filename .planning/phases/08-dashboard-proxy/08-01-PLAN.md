Criar `deploy/06-setup-dashboard.sh` seguindo fielmente o padrao de `deploy/05-setup-listener.sh` (per D-04). O script deve:

1. Header: `#!/bin/bash`, `set -euo pipefail`, echo banner "=== HelperTips Dashboard Setup ==="

2. Secao 1 — Unit file `/etc/systemd/system/helpertips-dashboard.service` (per D-01, D-05):
```ini
[Unit]
Description=HelperTips Dashboard
After=network.target postgresql.service
StartLimitBurst=5
StartLimitIntervalSec=300

[Service]
Type=simple
User=helpertips
Group=helpertips
WorkingDirectory=/home/helpertips
EnvironmentFile=/home/helpertips/.env
ExecStart=/home/helpertips/.venv/bin/gunicorn \
    --workers 2 \
    --bind 127.0.0.1:8050 \
    --timeout 120 \
    --access-logfile /var/log/helpertips/dashboard-access.log \
    --error-logfile /var/log/helpertips/dashboard-error.log \
    helpertips.dashboard:server
Restart=on-failure
RestartSec=60
StandardOutput=null
StandardError=null
NoNewPrivileges=true
PrivateTmp=true

[Install]
WantedBy=multi-user.target
```
Seguido de `chmod 644` no unit file.

NOTA CRITICA: `StartLimitBurst` e `StartLimitIntervalSec` em `[Unit]`, NAO em `[Service]` — pitfall documentado Phase 7.

3. Secao 2 — Logrotate: Sobrescrever `/etc/logrotate.d/helpertips` para incluir TANTO os logs do listener quanto do dashboard (per D-05). Conteudo:
```
/var/log/helpertips/listener.log {
    daily
    rotate 14
    compress
    delaycompress
    notifempty
    missingok
    create 0640 helpertips helpertips
}

/var/log/helpertips/dashboard-access.log
/var/log/helpertips/dashboard-error.log {
    daily
    rotate 14
    compress
    delaycompress
    notifempty
    missingok
    create 0640 helpertips helpertips
    postrotate
        systemctl kill -s USR1 helpertips-dashboard.service 2>/dev/null || true
    endscript
}
```

4. Secao 3 — Instalar gunicorn no venv:
```bash
echo ">>> Instalando gunicorn no venv..."
sudo -u helpertips /home/helpertips/.venv/bin/pip install -q "gunicorn>=25.0"
echo "gunicorn instalado"
```

5. Secao 4 — Criar diretorio de log (idempotente):
```bash
mkdir -p /var/log/helpertips
chown helpertips:helpertips /var/log/helpertips
```

6. Secao 5 — Recarregar systemd: `systemctl daemon-reload`

7. Secao 6 — Habilitar e iniciar servico (idempotente):
```bash
systemctl enable helpertips-dashboard
systemctl restart helpertips-dashboard
```

8. Secao 7 — Instrucoes finais com verificacao:
