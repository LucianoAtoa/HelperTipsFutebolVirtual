**deploy/01-provision-ec2.sh** — Script bash para executar no EC2 via SSH apos provisionamento. Conteudo exato:

```bash
#!/bin/bash
# Bootstrap inicial do EC2 para HelperTips
# Executar como root (ou com sudo) apos primeiro SSH
set -euo pipefail

echo "=== HelperTips EC2 Bootstrap ==="

# 1. Swap de 1GB (per D-01, t3.micro tem apenas 1GB RAM)
if [ ! -f /swapfile ]; then
    echo ">>> Criando swap de 1GB..."
    fallocate -l 1G /swapfile
    chmod 600 /swapfile
    mkswap /swapfile
    swapon /swapfile
    echo '/swapfile none swap sw 0 0' >> /etc/fstab
    echo "Swap criado com sucesso"
else
    echo "Swap ja existe, pulando"
fi

# 2. Usuario dedicado helpertips (nao root)
if ! id helpertips &>/dev/null; then
    echo ">>> Criando usuario helpertips..."
    useradd -m -s /bin/bash helpertips
    echo "Usuario helpertips criado"
else
    echo "Usuario helpertips ja existe, pulando"
fi

# 3. Pacotes base
echo ">>> Atualizando pacotes e instalando dependencias..."
apt update -y
apt install -y python3.12-venv git curl ca-certificates unzip

# 4. Diretorio de logs
mkdir -p /var/log/helpertips
chown helpertips:helpertips /var/log/helpertips

echo "=== Bootstrap concluido ==="
echo "Proximos passos: executar deploy/02-setup-postgres.sh"
```

**deploy/README-deploy.md** — Guia completo de provisionamento. Conteudo:

1. **Pre-requisitos**: Conta AWS ativa com billing habilitado, par de chaves SSH
2. **Passo 1 — Criar Key Pair**: No Console EC2 > Key Pairs > Create key pair, nome `helpertips-key`, tipo RSA, formato `.pem`. Salvar em `~/.ssh/helpertips-key.pem` e executar `chmod 400`
3. **Passo 2 — Criar Security Group**: Nome `helpertips-sg`, VPC default
   - Regra SSH: TCP 22, Source = IP pessoal /32 (usar https://checkip.amazonaws.com para descobrir IP)
   - Regra Dashboard: TCP 8050, Source = 0.0.0.0/0 e ::/0 (acesso temporario direto ate Phase 8 adicionar nginx)
   - Regra HTTP: TCP 80, Source = 0.0.0.0/0 e ::/0 (para Phase 8 nginx)
   - Saida: All traffic (padrao)
4. **Passo 3 — Lancar EC2**: AMI Ubuntu 24.04 LTS (buscar "Ubuntu" no launcher), t3.micro, key pair `helpertips-key`, SG `helpertips-sg`, EBS 20GB gp3
5. **Passo 4 — Alocar Elastic IP**: EC2 > Elastic IPs > Allocate > Associate to instance
6. **Passo 5 — Primeiro SSH e bootstrap**:
   ```
   ssh -i ~/.ssh/helpertips-key.pem ubuntu@ELASTIC_IP
   sudo bash deploy/01-provision-ec2.sh
   ```
7. **Verificacao**: `free -h` mostra swap de 1GB, `id helpertips` retorna info do usuario
8. **Nota sobre IP dinamico**: Se o ISP mudar o IP do desenvolvedor, atualizar a regra SSH no Console: EC2 > Security Groups > helpertips-sg > Inbound Rules > Edit

Per D-01: regiao us-east-1, instancia t3.micro, EBS 20GB gp3.
