# Phase 02.1 Plan 03: Query e ROI Complementares Summary

**One-liner:** `get_complementares_config` query JOIN + `calculate_roi_complementares` puro Python com Martingale por percentual.

## O Que Foi Feito

Implementacao completa das duas funcoes para suporte ao card ROI Complementares no dashboard, seguindo TDD rigoroso (RED → GREEN):

1. **`get_complementares_config(conn, mercado_slug)`** — Query com JOIN `complementares c JOIN mercados m`, filtrada por `m.slug = %s AND m.ativo = TRUE`, ordenada por `percentual DESC, slug ASC`. Retorna `list[dict]` via `dict(zip(columns, row))`.

2. **`calculate_roi_complementares(signals, complementares_config, stake, gale_on)`** — Calculo puro Python sem dependencia de DB ou Dash. Ignora sinais pendentes (`resultado=None`). Para cada sinal resolvido, itera sobre todos os complementares, chama `validar_complementar()` e acumula lucro/investido por mercado. Suporta Gale com `stake_base = stake * percentual`.
