  <implementation>
    Adicionar ao helpertips/queries.py:

    1. get_complementares_config(conn, mercado_slug: str) -> list[dict]:
       SQL:
       ```sql
       SELECT c.id, c.slug, c.nome_display, c.percentual, c.odd_ref, c.regra_validacao
       FROM complementares c
       JOIN mercados m ON c.mercado_id = m.id
       WHERE m.slug = %s AND m.ativo = TRUE
       ORDER BY c.percentual DESC, c.slug
       ```
       Retorna list[dict] via dict(zip(columns, row)).

    2. calculate_roi_complementares(signals, complementares_config, stake, gale_on) -> dict:
       - Iterar signals, ignorar pendentes (resultado=None)
       - Para cada sinal com resultado:
         - tentativa = signal.get("tentativa") or 1
         - Para cada complementar na config:
           - Se gale_on=False: stake_comp = stake * comp["percentual"]
           - Se gale_on=True: stake_comp acumulado = stake * (2^tentativa - 1) * comp["percentual"]; stake_comp winning = stake * 2^(tentativa-1) * comp["percentual"]
           - resultado_comp = validar_complementar(comp["regra_validacao"], signal.get("placar"), signal.get("resultado"))
           - Se resultado_comp == "GREEN": net = stake_comp_winning * (comp["odd_ref"] - 1) - prior_losses
           - Se resultado_comp == "RED": net = -investido_acumulado
           - Acumular por mercado (slug -> greens, reds, lucro, investido)
       - Retornar {"profit": total, "roi_pct": ..., "total_invested": ..., "por_mercado": [...]}

       A logica de Gale para complementares segue o MESMO padrao de calculate_roi() existente, so que o stake base e `stake * percentual` em vez de `stake`.

       ATENCAO: odd_ref vem do banco como Decimal (NUMERIC). Converter para float antes de calcular: `float(comp["odd_ref"])`, `float(comp["percentual"])`.
