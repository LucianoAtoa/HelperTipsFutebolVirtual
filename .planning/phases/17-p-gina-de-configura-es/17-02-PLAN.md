    FUNCAO _load_config() -> dict:
    - Abre conexao via get_connection()
    - Para cada slug em MERCADOS: chama get_mercado_config e get_complementares_config
    - Retorna dict com estrutura: {slug: {"mercado": dict, "complementares": list[dict]}}
    - Se mercado nao existe, usa defaults: stake_base=10.0, fator_progressao=2.0, max_tentativas=4

    LAYOUT (seguir UI-SPEC.md rigorosamente):
    - Titulo: html.H2("Configuracoes de Mercado", className="mt-4 mb-3")
    - dcc.Loading wrapping todo o conteudo
    - Bloco 1 — dbc.Row com 2 dbc.Col(lg=6), cada col com dbc.Card para um mercado:
      - dbc.CardHeader com nome do mercado (html.H5)
      - dbc.CardBody com 3 campos dbc.Input type="number":
        - id=f"input-stake-{slug}", label="Stake Base (R$)", value=stake_base, min=1, step=0.5
        - id=f"input-fator-{slug}", label="Fator de Progressao", value=fator_progressao, min=1, step=0.5
        - id=f"input-max-{slug}", label="Max. Tentativas", value=max_tentativas, min=1, max=10, step=1
    - Bloco 2 — Para cada mercado, dbc.Card com dbc.Table editavel de complementares:
      - Header: Complementar | Percentual (%) | Odd de Referencia
      - Cada linha: nome_display (texto), dbc.Input(id=f"input-pct-{slug}-{comp_id}", type="number", value=percentual*100, min=0, max=100, step=0.1), dbc.Input(id=f"input-odd-{slug}-{comp_id}", type="number", value=odd_ref, min=1, step=0.1)
      - NOTA: percentual no banco e fracao (0.20), no input exibir como porcentagem (20.0)
    - Bloco 3 — dbc.Card largura total "Preview em Tempo Real":
      - dbc.Row com 2 dbc.Col(lg=6):
        - Col 1: html.H5("Preview de Stakes por Tentativa") + dbc.Table com id=f"table-preview-{slug}" para cada mercado
        - Col 2: html.H5("Total em Risco por Tentativa") + dbc.Table com id=f"table-risco-{slug}" para cada mercado
    - Rodape: dbc.Button("Salvar Configuracoes", id="btn-salvar", color="primary", className="float-end mt-3")
    - dbc.Alert(id="alert-save", is_open=False, dismissable=True, className="mt-3")

    CALLBACKS (usar @callback, NAO @app.callback):

    1. Callback de preview (um por mercado) — Atualiza tabelas Preview e Risco:
       - Inputs: Input(f"input-stake-{slug}"), Input(f"input-fator-{slug}"), Input(f"input-max-{slug}"), + todos Input(f"input-pct-{slug}-{comp_id}") e Input(f"input-odd-{slug}-{comp_id}")
       - Outputs: Output(f"table-preview-{slug}", "children"), Output(f"table-risco-{slug}", "children")
       - Corpo: Chama calculate_preview_stakes e calculate_total_risco com valores dos inputs (sem banco)
       - Converte percentual de volta para fracao: input_value / 100.0
       - Gera dbc.Table rows com os resultados
       - CRITICO: Usar funcao auxiliar _build_preview_table(preview_data) e _build_risco_table(risco_data) para gerar as tabelas

    2. Callback de salvar:
       - Inputs: Input("btn-salvar", "n_clicks")
       - States: Todos os inputs de stake/fator/max + pct/odd de cada complementar
       - Outputs: Output("alert-save", "children"), Output("alert-save", "color"), Output("alert-save", "is_open"), Output("btn-salvar", "disabled")
       - Corpo:
         ```python
         if not n_clicks:
             return no_update, no_update, no_update, no_update
         try:
             conn = get_connection()
             for slug in MERCADOS:

                 comps_to_save = [{"id": comp_id, "percentual": pct_value / 100.0, "odd_ref": odd_value} ...]
                 save_complementares_config(conn, comps_to_save)
             conn.close()
             return "Configuracoes salvas com sucesso.", "success", True, False
         except Exception as e:
             return f"Erro ao salvar. Verifique a conexao com o banco e tente novamente.", "danger", True, False
         ```
