### Issues Encontrados (Não-Fatais)

1. **Cláusula OR duplicada** — `store.py` linhas 55-56: condição WHERE duplicada no upsert. Funcionalmente correto (OR idempotente). Afeta: DB-02, DB-03.

2. **Race window conn=None** — `listener.py`: entre `client.start()` (l.93) e `conn = get_connection()` (l.144), handlers já registrados podem disparar com `conn=None` se mensagem chegar durante `selecionar_grupo()`. Probabilidade baixa. Afeta: LIST-01, LIST-02, DB-04.

3. **Exception catch estreito** — `listener.py`: retry loop captura apenas `ConnectionError`, não `RPCError` do Telethon. `FloodWaitError` handling é explicitamente v2 (RESIL-03). Afeta: LIST-05.

4. **Dead export** — `queries.py` l.35: re-export de `get_connection` nunca consumido (todos importam direto de `helpertips.db`).

5. **ANAL-03 vazio** — `get_winrate_by_periodo` wired mas parser sempre retorna `periodo: None`. Dashboard trata graciosamente ("Sem dados"). Design intencional documentado.
